When new players hear things like "Base set is a p2 win with optimal play from both sides", they often get very frustrated and think that Lunarch should do something about this, make it more balanced. But the truth is that this problem is inevitable in any turn-based deterministic game. To see why, imagine a tree of every possible game state. Each state where the winner is decided is marked as a winning position for that player. For each position that comes directly before a decided position, look at all of the possible following positions, and if any of them is a win for the player who's turn it is at the current position, then this one is also a win for that player (because optimal play means they can get it to a winning position from here). Otherwise it's a win for the other player. You can assign a theorical optimal-play victor to every possible Prismata gamestate this way, and therefore, even at the start of the game, there is always going to be one player - p1 or p2 - who can already force a win regardless of how the other person plays. It works for any turn-based deterministic game - tic-tac-toe, connect four, checkers, chess, go, whatever. They're all autowins (or auto-draws).

So, two potential criticisms remain: one that Prismata sets should all be autodraws instead of autowins, and the other that turn-based determinstic strategy games are simply fundamentally flawed. I quite like turn-based deterministic strategy games (it's the only way to do a real pure strategy game), so I'm not going to give up the genre just because of this one issue. The other criticism I concede is valid. Prismata is not a perfect game. However, in most sets it only takes a very small mistake to make the theorically-winning player lose to optimal play from the theorically-losing player, and it has a high skill ceiling (ie. playing perfectly for an entire match is very difficult). So log in, hop in queue and let's enjoy this flawed but amazing game together.
